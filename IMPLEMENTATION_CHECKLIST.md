# Checklist poprawek po audycie DOCX

Założenie: poprawki punktowe, bez dużego reworku graficznego. Po każdej zmianie sprawdzić, czy treści nadal układają się spójnie w całej stronie.

## Do wdrożenia

- [x] Naprawić liczniki, które czasami pokazują `0`.
- [x] Usunąć sticky bar na dole strony mobilnej.
- [x] Usunąć ogólną mgłę/rozmycie na jasnym motywie.
- [x] Powiększyć drobne fonty i podbić kontrast, szczególnie w kalkulatorze.
- [x] Ukryć sekcję opinii oraz social proof z hero.
- [x] Zrobić logotypy firm kolorowe domyślnie.
- [x] Na mobile pokazać firmy/loga jako poziomy scroll zamiast siatki boxów.
- [x] Poprawić mobile spacing: przyciski, ikony, teksty i odstępy.
- [x] Zmniejszyć zbyt puste przestrzenie między sekcjami.
- [x] Poprawić CTA/hero, żeby obrazy i tekst nie wyglądały na przydymione.
- [x] W mapie zmienić tekst powrotu na `Powrót`.
- [x] W mapie zostawić widoczny nagłówek, nie chować go.
- [x] Poprawić widoczność suwaków i kontrolek.
- [x] Poprawić galerię/przełączanie billboardu tak, żeby wyglądało jak zmiana grafiki na tablicy.
- [x] Poprawić widoczność tabeli w ciemnym motywie.
- [x] Uruchomić lint/build i sprawdzić stronę w przeglądarce.
